# CardDB Unreal Importer Manifest Suite Report

## Purpose

Added a test-only CardDB importer manifest suite aggregator.

The suite helper validates a named set of manifest fixture references, evaluates each manifest through the existing manifest helper, and emits one deterministic suite-level readiness report. It is not a production importer, production loader, production zone model, runtime activation path, gameplay rule, schema migration system, or EffectRunner path.

## Suite Schema

Required suite fields:

- `suite_schema_version`
- `suite_id`
- `manifests`

Optional suite fields:

- `description`
- `metadata`

`suite_schema_version` must equal `1`. Empty `manifests` is invalid for importer planning.

## Manifest Entry Schema

Required manifest entry fields:

- `name`
- `path`

Optional manifest entry fields:

- `metadata`

Manifest names must be unique within the suite. Duplicate paths are allowed when aliases differ, because a suite may intentionally compare the same manifest fixture under different test names.

Manifest entry order is preserved in validation and export.

## Path Safety Policy

Manifest paths are relative to:

```text
Reference/GodotCanon/CardDBSchemaFixtures/
```

Paths must not be absolute, include drive letters, include `..` traversal, escape the fixture root, target `Reference/GodotProject`, or point to missing files.

Unsafe paths fail closed with `suite_manifest_path_invalid`. Missing files fail with `suite_manifest_path_not_found`.

## Evaluation Behavior

If suite validation fails, no manifest evaluation runs.

If suite validation succeeds, each manifest entry is evaluated through `FWBCardDBImporterManifestForTests`.

Suite evaluation can be `ok=true` while a valid manifest contains not-ready bundles. Not-ready bundles remain batch-readiness results, not suite validation failures.

## Aggregate Counts

Suite evaluation records:

- total manifests
- valid manifests
- invalid manifests
- total batches
- total bundles
- ready bundles
- not-ready bundles

Manifest entries in the export include alias, ok state, batch count, total bundle count, ready bundle count, and not-ready bundle count.

## Aggregate Diagnostic Summary

The suite helper gathers all bundle readiness results from evaluated manifest batch results and builds one `FWBCardDBImporterDiagnosticSummaryForTest`.

Invalid suite validation does not build readiness summaries. Manifest validation failures are represented as manifest-level status, not as readiness summary diagnostics.

## Export JSON Shape

Suite snapshots export deterministic condensed JSON:

```json
{
  "ok": true,
  "reason": "",
  "suite_id": "fixture_suite_core",
  "manifest_count": 2,
  "valid_manifests": 2,
  "invalid_manifests": 0,
  "total_batches": 3,
  "total_bundles": 8,
  "ready_bundles": 6,
  "not_ready_bundles": 2,
  "manifests": [],
  "summary": {
    "reason_summaries": [],
    "diagnostic_summaries": []
  },
  "diagnostics": []
}
```

Exports do not include full manifest exports, full batch exports, full bundle readiness exports, full source JSON, diagnostic messages, payload bodies, card text, or hidden values.

## Hidden-Token Safety

Hidden tokens in suite ids, manifest aliases, descriptions, metadata strings, or paths fail hidden-info policy.

Suite diagnostics and exports sanitize hidden values. The hidden-token fixture verifies `SECRET` does not appear in suite export output.

## Helper Relationships

The suite helper calls the manifest helper after suite validation succeeds.

The manifest helper validates batch references and calls the batch-readiness helper.

The batch-readiness helper calls the readiness helper and uses the diagnostic-summary helper.

The suite helper does not parse production data, load runtime CardDB definitions, create zones, execute effects, mutate game state, generate activation candidates, or generate activation legal actions.
