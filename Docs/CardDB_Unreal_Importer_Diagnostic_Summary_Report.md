# CardDB Unreal Importer Diagnostic Summary Report

## Purpose

Added a test-only CardDB importer diagnostic summary helper.

The helper groups readiness results for future importer planning. It is not a production importer, production loader, production diagnostic framework, zone model, runtime activation path, gameplay rule, or EffectRunner path.

## Summary Fields

`FWBCardDBImporterDiagnosticSummaryForTest` contains:

- `TotalBundles`
- `ReadyBundles`
- `NotReadyBundles`
- `ReasonSummaries`
- `DiagnosticSummaries`
- `ReadyBundleSourcePaths`
- `NotReadyBundleSourcePaths`

The export shape is deterministic condensed JSON.

## Reason Grouping

Not-ready readiness results are grouped by stable `Reason`.

Empty reasons are normalized to:

```text
unknown
```

Reason summaries are sorted by reason ascending.

## Diagnostic Code Grouping

Bundle diagnostics from readiness results are grouped by `DiagnosticCodeToString`.

Diagnostic summaries are sorted by diagnostic code ascending.

Diagnostic messages are not exported.

## Affected Bundle And Card Counts

For each diagnostic code:

- `count` is the total number of diagnostic occurrences
- `affected_bundle_count` is the number of distinct readiness results with that code
- `affected_card_count` is the number of distinct source-path plus card-index contexts where `CardIndex >= 0`

The summary does not export card ids or effect ids.

## Source Path Sanitization

Source paths use clean filenames only.

If a clean filename contains a hidden-token pattern, the exported path becomes:

```text
redacted_source
```

Ready and not-ready source path lists are sorted ascending.

## Hidden-Token Safety

Summary output includes only stable reason tokens, diagnostic code tokens, aggregate counts, and sanitized clean filenames.

It omits:

- full source JSON
- diagnostic messages
- card ids
- effect ids
- public labels/text
- payload bodies
- hidden referenced values

Hidden-token summary coverage verifies `SECRET` does not appear in summary output.

## Expected Summary Export Fixtures

Expected summary snapshots cover:

- mixed ready/not-ready bundles
- schema failure diagnostics
- card-context affected counts
- all-ready bundles
- hidden-token-safe failure
- empty input

Expected exports live under:

```text
Reference/GodotCanon/CardDBSchemaFixtures/ExpectedExports/
```

## Relationship To Importer Readiness

The summary helper consumes `FWBCardDBImporterReadinessResultForTest` values.

It does not evaluate bundles itself, parse production data, load card definitions, or mutate state. Readiness remains the validation boundary; summaries provide aggregate planning output.

## Relationship To Batch Readiness

The batch-readiness helper now uses diagnostic summaries for grouped output across named bundle sets.

Batch snapshots include the same reason and diagnostic summary arrays used by this helper, while leaving detailed per-bundle diagnostics as diagnostic code strings only.
