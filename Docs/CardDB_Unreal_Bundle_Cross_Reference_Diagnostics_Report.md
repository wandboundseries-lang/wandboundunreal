# CardDB Unreal Bundle Cross-Reference Diagnostics Report

## Purpose

Added test-only cross-reference diagnostics for CardDB bundle schema fixtures.

This remains fixture validation only. It is not a production CardDB importer, loader, zone model, runtime activation path, gameplay rule, or EffectRunner path.

## Reference Schema

Card-level references:

```json
{
  "references": {
    "card_ids": ["other_card_id"]
  }
}
```

Effect-level references:

```json
{
  "references": {
    "card_ids": ["other_card_id"],
    "effect_refs": [
      {
        "card_id": "other_card_id",
        "effect_id": "some_effect"
      }
    ]
  }
}
```

Payload-level references:

```json
{
  "references": {
    "card_ids": ["other_card_id"]
  }
}
```

The `references` object is authoring/test metadata only. Root-card validation checks shape. Bundle validation resolves references against the bundle card/effect index. No `references` data maps into `FWBCardDefinition`.

## Allowed Fields

Strict mode now allows `references` on:

- card objects
- activated effect objects
- payload objects

Allowed `references` fields:

- `card_ids`
- `effect_refs`
- `metadata`

Allowed `effect_refs[]` fields:

- `card_id`
- `effect_id`
- `metadata`

Allowed `references.metadata` fields:

- `notes`
- `test_only`

Unknown reference fields fail in strict mode with `unknown_reference_field`.

## Diagnostics

Added stable diagnostic strings:

- `missing_card_reference`
- `missing_effect_reference`
- `reference_malformed`
- `unknown_reference_field`

Malformed references fail closed before any production import exists. Missing references are resolved only at bundle scope.

## Bundle Resolution Policy

Bundle validation builds a card id index and per-card effect id index.

If duplicate card ids are present, bundle validation preserves `card_id_duplicate` diagnostics and skips cross-reference resolution because the reference index is ambiguous.

Reference resolution order is deterministic:

- card-level `references`
- effect-level `references`
- payload-level `references`

Stable JSON paths include:

- `$.cards[i].references.card_ids[j]`
- `$.cards[i].activated_effects[e].references.card_ids[j]`
- `$.cards[i].activated_effects[e].references.effect_refs[r]`
- `$.cards[i].activated_effects[e].payloads[p].references.card_ids[j]`

## Hidden-Token Safety

Missing-reference diagnostics use generic messages and safe context only. They do not echo referenced ids, hidden tokens, or full JSON snippets.

Fixture coverage verifies `SECRET` does not appear in diagnostic message, card id, bundle card id, effect id, or JSON path.
