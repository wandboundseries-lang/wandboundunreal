# CardDB Unreal Bundle-Level Export Diagnostics Report

## Purpose

Added test-only expected export snapshots for malformed CardDB bundle-level diagnostics.

This remains fixture validation only. It is not a production CardDB importer, loader, zone model, runtime activation path, gameplay rule, or EffectRunner path.

## Bundle-Level Export Fixture Coverage

Expected export snapshots now cover:

- missing `bundle_schema_version`
- unsupported `bundle_schema_version`
- missing `carddb_version`
- missing `cards`
- malformed `cards`
- empty `cards`
- duplicate card ids
- bundle containing an invalid card
- strict unknown top-level bundle field
- strict unknown bundle metadata field
- non-strict unknown bundle field allowed
- strict validation of the non-strict fixture
- mixed bundle-level and card-level errors
- hidden-token-safe missing reference diagnostics

Expected exports live under:

```text
Reference/GodotCanon/CardDBSchemaFixtures/ExpectedExports/
```

## Exported Diagnostic Behavior

Export JSON remains deterministic and condensed.

Bundle-level diagnostics export `card_index = -1`, empty `card_id`, empty `effect_id`, and a stable bundle-level `json_path`.

Card-level diagnostics preserve card index, safe card id, safe effect id when available, and inferred card/effect/payload JSON paths.

Diagnostic messages, full source JSON, public labels, public text, payload bodies, and referenced hidden values are not exported.

## Duplicate Card Id Export Policy

Duplicate card id exports use:

- `code = "card_id_duplicate"`
- the second duplicate `card_index`
- the safe duplicate card id
- `json_path = "$.cards"`
- empty dependency order

## Strict And Non-Strict Export Policy

Default validation remains non-strict, so `non_strict_bundle_unknown_field_allowed.json` exports `ok = true` and no diagnostics.

Strict validation of the same fixture exports `unknown_top_level_field` with `json_path = "$"`.

Strict unknown bundle metadata fields export `unknown_metadata_field` with `json_path = "$.metadata"`.

## Mixed Diagnostic Order Policy

Mixed bundle/card snapshots preserve current validation order:

1. bundle-level strict unknown-field diagnostics
2. card-level diagnostics in `cards[]` order

The mixed fixture snapshots `unknown_top_level_field` before `effect_id_missing`.

## Hidden-Token Export Safety

The hidden reference snapshot verifies that a referenced hidden token is not echoed in export JSON.

Missing reference diagnostics export generic owner context only and do not include the missing referenced id.
