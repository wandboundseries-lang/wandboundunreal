# CardDB Unreal Importer Manifest Report

## Purpose

Added a test-only CardDB importer manifest validator.

The validator groups named batch-readiness sets, validates manifest structure before evaluation, and exports deterministic manifest-level snapshots. It is not a production importer, production loader, production zone model, runtime activation path, gameplay rule, schema migration system, or EffectRunner path.

## Manifest Schema

Required manifest fields:

- `manifest_schema_version`
- `manifest_id`
- `batches`

Optional manifest fields:

- `description`
- `metadata`
- `default_compatibility`

`manifest_schema_version` must equal `1`. Empty `batches` is invalid for importer planning.

## Batch Schema

Required batch fields:

- `batch_name`
- `bundles`

Optional batch fields:

- `description`
- `metadata`
- `compatibility`

Batch names must be unique within the manifest. Batch order is preserved in validation and export.

## Bundle Entry Schema

Required bundle entry fields:

- `name`
- `path`

Optional bundle entry fields:

- `metadata`
- `compatibility`

Bundle names must be unique within a batch. Bundle entry order is preserved in validation and export.

## Compatibility Override Behavior

Compatibility can be declared at three levels:

- manifest `default_compatibility`
- batch `compatibility`
- bundle entry `compatibility`

Each lower level inherits the previous level and overrides only fields it declares.

Accepted compatibility fields:

- `enable`
- `require_source_version`
- `target_source_version`
- `directly_supported_source_versions`
- `supported_transitions`

Malformed compatibility fields fail closed with `manifest_compatibility_malformed`.

## Path Safety Policy

Bundle paths are relative to:

```text
Reference/GodotCanon/CardDBSchemaFixtures/
```

Paths must not be absolute, include drive letters, include `..` traversal, escape the fixture root, target `Reference/GodotProject`, or point to missing files.

Unsafe paths fail closed with `manifest_bundle_path_invalid`. Missing files fail with `manifest_bundle_path_not_found`.

## Evaluation Behavior

If manifest validation fails, no batch readiness evaluation runs.

If manifest validation succeeds, each manifest batch is evaluated through `FWBCardDBImporterBatchReadinessForTests`.

Manifest evaluation can be `ok=true` while some bundles are not ready. Not-ready bundles remain batch-readiness results, not manifest validation failures.

## Export JSON Shape

Manifest snapshots export deterministic condensed JSON:

```json
{
  "ok": true,
  "reason": "",
  "manifest_id": "fixture_manifest_core",
  "batch_count": 1,
  "batches": [],
  "diagnostics": []
}
```

Valid manifest exports include batch-level summaries only. They do not include full per-bundle readiness exports.

Invalid manifest exports include manifest diagnostics only.

## Hidden-Token Safety

Hidden tokens in manifest ids, batch names, bundle names, descriptions, metadata strings, paths, or compatibility strings fail hidden-info policy.

Manifest diagnostics and exports sanitize hidden values. The hidden-token fixture verifies `SECRET` does not appear in manifest export output.

## Helper Relationships

The manifest helper calls the batch-readiness helper only after manifest validation succeeds.

The batch-readiness helper calls the readiness helper for each bundle and uses the diagnostic-summary helper for grouped batch output.

The manifest helper does not parse production data, load runtime CardDB definitions, create zones, execute effects, mutate game state, generate activation candidates, or generate activation legal actions.

## Suite Integration

Importer manifest suites now group multiple manifest fixtures for test-only evaluation.

The suite helper validates duplicate manifest aliases, safe relative manifest paths, and metadata before calling this manifest helper. Suite exports include manifest-level counts and aggregate readiness summaries only, not full manifest exports.
