# CardDB Unreal Bundle Diagnostic Reporting Report

## Purpose

Added stable per-card diagnostic context to the test-only CardDB bundle schema validator.

This remains fixture validation only. It is not a production CardDB importer, loader, zone model, or runtime activation path.

## Diagnostic Context Fields

`FWBCardDBSchemaValidationDiagnostic` now includes optional bundle context:

- `CardIndex`: zero-based index inside `cards[]`, or `-1` outside bundle card validation.
- `BundleCardId`: safe `card_id` from the bundle card object when available.
- `EffectId`: existing effect id context, preserved when safely available.
- `JsonPath`: stable path-like location such as `$.cards[1].activated_effects[0].payloads[0]`.

Root-card validation outside bundle mode keeps `CardIndex = -1` and no bundle card id.

## Diagnostic Ordering Policy

Ordering is deterministic:

- bundle-level diagnostics are emitted before card-level diagnostics when discovered before `cards[]` validation.
- card diagnostics are aggregated in `cards[]` order.
- diagnostics within a card preserve the existing root-card validator order.
- strict unknown-field diagnostics preserve sorted field-name behavior.

## Duplicate Card Id Context Policy

Duplicate card ids fail with `card_id_duplicate`.

The duplicate diagnostic records:

- `CardIndex` as the second duplicate index
- `BundleCardId` as the duplicate id when safe
- `JsonPath` as `$.cards`

## Hidden-Token Diagnostic Safety

Context fields do not include full JSON snippets or hidden values.

Bundle context sanitizes unsafe card/effect ids before copying them into diagnostics. Hidden-token fixture coverage verifies diagnostic message, card id, bundle card id, effect id, and JSON path do not leak `SECRET` tokens.

## Multi-Card Fixture Coverage

Added fixtures covering:

- multiple invalid cards with stable per-card context
- effect and payload path context
- duplicate card id context
- bundle-level diagnostics before card-level diagnostics
- hidden-token diagnostic safety
- valid bundle with no diagnostics

These helpers are part of the test-only validator surface under `WandboundTests`.
