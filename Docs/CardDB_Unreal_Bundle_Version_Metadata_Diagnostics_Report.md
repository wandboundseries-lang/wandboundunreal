# CardDB Unreal Bundle Version Metadata Diagnostics Report

## Purpose

Added test-only CardDB bundle diagnostics for source versioning and bundle metadata policy.

This remains fixture validation only. It is not a production CardDB importer, loader, zone model, runtime activation path, gameplay rule, or EffectRunner path.

## `carddb_version` Policy

`carddb_version` is required.

Missing `carddb_version` keeps the existing `carddb_version_missing` diagnostic.

Present values must be non-empty strings, max 64 characters, using only letters, numbers, underscore, hyphen, and dot.

Invalid present values fail with:

```text
invalid_carddb_version
```

## `source_version` Policy

`source_version` is optional.

When present, it must be a string, max 64 characters, using only letters, numbers, underscore, hyphen, dot, and slash.

Invalid values fail with:

```text
invalid_source_version
```

## `migration_notes` Policy

`migration_notes` is optional.

When present, it must be a string and max 512 characters.

Malformed values fail with:

```text
migration_notes_malformed
```

Hidden tokens in migration notes fail with:

```text
hidden_info_policy_violation
```

## Metadata Object Policy

Bundle `metadata` is optional.

When present, it must be an object. Allowed fields remain:

- `author`: string, max 64
- `notes`: string, max 512
- `source`: string, max 128
- `version`: string, max 64
- `test_only`: boolean

Malformed metadata shape or value types fail with:

```text
metadata_malformed
```

## Hidden-Token Safety

Hidden tokens in migration notes or metadata string values fail closed with `hidden_info_policy_violation`.

Export snapshots include only diagnostic code and JSON path context. They do not include hidden source values or full metadata values.

## Strict-Mode Metadata Behavior

Strict mode still rejects unknown metadata fields with:

```text
unknown_metadata_field
```

The strict unknown-field diagnostic remains separate from `metadata_malformed`.

## Expected Export Coverage

Expected export snapshots now cover:

- valid full version/metadata bundle
- invalid CardDB version values
- invalid source version values
- malformed migration notes
- migration notes hidden token
- malformed metadata object/value types
- metadata hidden token
- strict unknown metadata field
