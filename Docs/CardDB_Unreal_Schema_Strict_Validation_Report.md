# CardDB Unreal Schema Strict Validation Report

## Purpose

Added explicit strict unknown-field validation to the test-only CardDB schema fixture validator.

The validator still lives under `WandboundTests` and remains a fixture contract tool, not a production CardDB importer, loader, zone model, or runtime activation system.

## Validation Options

`FWBCardDBSchemaValidationOptions` now controls strict validation:

```cpp
bool bStrictUnknownFieldRejection = false;
```

Default validator behavior remains non-strict. Existing fixture calls that omit options continue to allow unknown authoring fields unless another existing diagnostic catches malformed data, hidden-info text, or bad player-facing labels.

## Strict Unknown-Field Diagnostics

Strict mode can emit:

- `unknown_top_level_field`
- `unknown_card_field`
- `unknown_effect_field`
- `unknown_source_gate_field`
- `unknown_cost_gate_field`
- `unknown_payload_field`
- `unknown_metadata_field`

Existing diagnostic strings were not renamed.

Bundle validation also uses `unknown_top_level_field` for unknown bundle top-level fields and `unknown_metadata_field` for unknown bundle metadata fields.

## Allowed Field Sets

Strict mode allow-lists:

- top-level/card fields such as `schema_version`, `card_id`, `public_name`, `kind`, `stats`, and `activated_effects`
- stats fields such as `hp`, `max_hp`, `atk`, `ar`, `base_rl`, `current_rl`, and `rr`
- effect fields such as `effect_id`, `public_label`, `target_requirement`, `source_gate`, `payloads`, and `metadata`
- source gate fields such as `required_zone`, `timing`, source ownership gates, stun/frozen gates, cost gate, usage gate, and `metadata`
- cost gate fields such as external affordability, RR requirement, cost kind, and `metadata`
- payload fields for the current supported generic payload families
- bundle top-level fields such as `bundle_schema_version`, `carddb_version`, `source_version`, `migration_notes`, `metadata`, and `cards`

Unknown fields inside `stats` fail with `unknown_card_field`.

## Metadata Policy

`metadata` is allowed only where explicitly listed.

Allowed metadata fields are intentionally narrow:

- card metadata: `author`, `notes`, `source`, `version`, `test_only`
- effect metadata: `notes`, `source`, `test_only`
- source gate metadata: `notes`, `test_only`
- cost gate metadata: `notes`, `test_only`
- payload metadata: `notes`, `test_only`
- bundle metadata: `author`, `notes`, `source`, `version`, `test_only`

Unknown fields inside metadata fail with `unknown_metadata_field` in strict mode.

## Fail-Closed Behavior

Strict mode appends deterministic unknown-field diagnostics in sorted field-name order. Any diagnostic keeps `FWBCardDBSchemaValidationResult::bOk` false.

Diagnostics do not include full JSON snippets.
